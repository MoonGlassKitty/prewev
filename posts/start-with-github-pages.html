把 prewev 挂到 GitHub Pages

从本地静态文件到线上博客，记录一次真实部署，也给之后的页面更新留一条路。

这个站点起点很轻：`index.html`、`styles.css`、`script.js`，再加几篇静态文章。目录名叫 `prewev`，后来被推到 GitHub 仓库，变成可以直接访问的个人实验笔记。

先让它能被访问

GitHub Pages 会把仓库里的静态文件发布到网页上。这个项目用 `main` 分支根目录发布， 不需要构建工具，也不需要服务器。

对 MoonGlassKitty 来说，先把站点跑起来，比把模板想得很完美更重要。

再让它适合长期写

后面每次改完页面，只要提交并推送，GitHub Pages 就会自动更新。这个流程足够简单，适合继续写项目复盘。

  • 主页负责展示最近实验和个人介绍。
  • 文章放在 posts/ 目录里。
  • logo、封面图和项目图片放在 assets/ 目录里。
