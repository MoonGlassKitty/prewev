从一个静态网页开始个人博客

不用复杂框架，也可以用 GitHub Pages 开始持续写作。先跑起来，再慢慢打磨。

个人博客最容易卡住的地方，往往不是技术，而是想一次做得太完整。静态网页的好处是边界很清楚： 一个入口文件、一份样式、几篇文章，就能公开发布。

先让它能被访问

GitHub Pages 会把仓库里的静态文件发布到网页上。只要根目录里有 index.html，并且仓库开启 Pages，访问地址就会自动生成。

一个可以维护的网站，比一个永远停在计划里的完美网站更有价值。

再让它适合长期写

当文章变多时，可以逐步增加归档页、分类页、RSS 或静态站点生成器。开始阶段保持简单，反而更容易养成更新节奏。

  • 主页展示最近文章和个人介绍。
  • 每篇文章放在 posts/ 目录里。
  • 图片和其他资源放在 assets/ 目录里。
